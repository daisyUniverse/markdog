# barkdown
This will be where I will keep all notes/implementations of my attempt at reverse engineering and reimplementing the dialogue parsing system from Deltarune

## Goal:
 0. Study the code, and work out how dialogue entries are created, invoked, and parsed - Also document this heavily - I want the spec to be clearly defined
 1. Create a functional intermediary format that can contain every piece of data required to perfectly recreate all possible Deltarune dialogues
 2. Manually recreate some complicated dialogue trees from the game
 3. Create a parser that can correctly interpret these the same as the game would
 4. Create a frontend that can render these entries
 5. Create a script that can read entries from the game and translate them to my intermediary format
 6. Create a database of all dialogue by scanning the game, and ensure that they are parsed as intended, fix whatevers broken
 7. Create a website which will allow users to browse through and read entries
 8. Create both an API and a UI for creating and retrieving custom dialogue trees
 9. Make it so that it can be embedded in a website as an x-frame


## Feature Creep
- Allow users to draw out their own faces/portraits/emotions in a built in simple and fun paint tool
- Allow users to build rooms out of DR assets which can have flavor text attatched to whatever you want
- Allow these rooms to have doorways that can link to other rooms by ID
- Allow custom branching conversations to branch off into other conversations by linking the ID as the next message
